# Getting started

Install pimm, run your first job, and learn the handful of ideas that the rest
of the documentation assumes.

- {doc}`What is pimm? <overview>` - plain-language intro: what pimm is and whether it fits your detector data.
- {doc}`Installation <installation>` - container (recommended) or from source. CUDA, FlashAttention, and the local extension builds.
- {doc}`Quickstart <quickstart>` - your canonical first run: the one command (`pimm launch`), a 60-second sanity check, a real run, dry-runs, resume, export, and the full CLI surface. Start here.
- {doc}`How pimm works <concepts>` - packed tensors, registries, Python configs, and the launch layers - the mental model.
