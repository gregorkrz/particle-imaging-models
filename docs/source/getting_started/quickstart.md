# Quickstart

This page gets you from a fresh checkout to a real run. It assumes you have
{doc}`installed pimm <installation>` and have PILArNet-M data (or are happy to
use tiny `max_len` limits for a quick check).

## Build your run command

Pick your target, resources, and config — the matching `pimm launch` / `pimm
submit` command assembles below, ready to copy.

## The one command you need

The primary entry point for local GPU(s) is `pimm launch`:

```bash
pimm launch --train.config <config-path> [-- TRAIN_OVERRIDES...]
```

`--train.config` is a path under `configs/`, with or without `.py`. Everything
after a bare `--` separator is a training override, written as `KEY=VALUE`.

```text
pimm launch  ─▶  scripts/train.sh  ─▶  torchrun  ─▶  pimm/train.py (DefaultTrainer)
                     │
                     ├─ snapshot code  → exp/<dataset>/<name>/code/
                     ├─ resolve config + overrides
                     └─ pick resume checkpoint (if --train.resume)
```

`pimm launch` composes a launch script and runs `scripts/train.sh`, which
prepares experiment paths, snapshots the code, resolves checkpoints, and calls
`torchrun`.

## 60-second sanity check

Tiny limits, no W&B, no workers — verifies the whole pipeline end to end:

```bash
pimm launch \
  --train.config panda/semseg/semseg-pt-v3m2-pilarnet-ft-5cls-enc-upcast-fft \
  --run.name smoke \
  -- epoch=1 data.train.max_len=64 data.val.max_len=32 \
     batch_size=4 num_worker=0 use_wandb=False
```

This writes everything under `exp/panda/semseg/smoke/`.

## A real single-GPU run

```bash
pimm launch \
  --train.config panda/semseg/semseg-pt-v3m2-pilarnet-ft-5cls-enc-upcast-fft \
  --run.name semseg-pt-v3m2 \
  --resources.nproc-per-node 1
```

Multi-GPU on one node needs no Slurm — `train.sh` uses `torchrun`:

```bash
pimm launch \
  --train.config panda/semseg/semseg-pt-v3m2-pilarnet-ft-5cls-enc-upcast-fft \
  --resources.nproc-per-node 4
```

See {doc}`../distributed/index` for how batch sizes and workers are split across
GPUs.

## Launcher flags you'll actually use

CLI flags are generated by [Tyro](https://github.com/brentyi/tyro) from typed
dataclasses, so they are **nested and dotted**. There are no flat `--config` or
`--set` flags.

| Flag | Description |
|------|-------------|
| `--train.config` | Config path under `configs/`, with or without `.py` |
| `--run.name` | Experiment name (default: `<config>-<timestamp>`) |
| `--resources.nproc-per-node` | Torchrun processes (GPUs) per node |
| `--resources.nnodes` | Number of nodes |
| `--train.weight` | Path / `hf://` URI to a checkpoint to warm-start from |
| `--train.resume` | Resume from the latest complete checkpoint |
| `--train.no-code-copy` | Skip the code snapshot, run from repo source (dev mode) |
| `--dry-run` | Render the command/script without executing |
| `--recipe PATH` | Load a saved launch recipe (see {doc}`../hpc/index`) |

:::{warning}
Training overrides after `--` must be bare `KEY=VALUE` tokens. Any token
starting with `--` is rejected. `-- --options epoch=10` is **invalid** for
`pimm launch`/`pimm submit` — write `-- epoch=10`. (The `-- --options ...` form
is only for direct `scripts/train.sh` calls.)
:::

## Overriding config values

Two ways, smallest mechanism first:

::::{tab-set}

:::{tab-item} CLI override (quick probes)
```bash
pimm launch --train.config panda/pretrain/pretrain-sonata-v1m1-pilarnet-smallmask \
  -- epoch=50 data.train.max_len=500000 optimizer.lr=3e-5
```
Dotted keys patch nested config; values are parsed as int → float → bool →
list/tuple → string. Great for quick checks and short-lived probes.
:::

:::{tab-item} Child config (reusable variants)
```python
# configs/panda/pretrain/my_variant.py
_base_ = ["./pretrain-sonata-v1m1-pilarnet-smallmask.py"]

epoch = 100
data = dict(train=dict(max_len=100000), val=dict(max_len=1000))
optimizer = dict(lr=3e-5, weight_decay=0.2)
```
Use a child config for anything you want to review, resume, or compare. See
{doc}`../configuration/index`.
:::

::::

## Always dry-run before something expensive

```bash
pimm launch --train.config panda/pretrain/pretrain-sonata-v1m1-pilarnet-smallmask \
  --resources.nproc-per-node 4 --dry-run
```

The rendered `sh scripts/train.sh ... -c <config> ... --options ...` is the
authoritative record of the config path, run name, resources, and overrides.

## What a run produces

By default the launcher snapshots the codebase and trains from that copy for
reproducibility:

```text
exp/<dataset>/<name>/
  code/                 # snapshot of pimm, scripts, tools (run trains from here)
  config.py             # resolved config after inheritance + overrides + seed
  resolved_config.json  # full resolved config as JSON
  model_config.json     # cfg.model as JSON
  run_metadata.json     # command, host, git status, original config path
  train.log             # training log
  model/                # checkpoints (see Checkpoints section)
```

Set `MODEL_DIR` to put the (large) `model/` directory on another disk; the
experiment `model/` becomes a symlink to it.

## Resume

Resuming reuses the experiment's **saved** `config.py`, not the original file
under `configs/`:

```bash
pimm launch \
  --train.config panda/pretrain/pretrain-sonata-v1m1-pilarnet-smallmask \
  --run.name semseg-pt-v3m2 \
  --train.resume
```

The experiment directory must already exist; the launcher selects the newest
complete checkpoint (`last`, `last.prev`, `model_last.pth`). Resume is
*exact* — RNG, dataloader position, step, and optimizer state are all restored,
even mid-epoch. See {doc}`../checkpoints/index` and {doc}`../hpc/resuming`.

## Logging

Rank 0 writes either Weights & Biases or TensorBoard:

```bash
export WANDB_API_KEY=...
pimm launch --train.config panda/pretrain/pretrain-sonata-v1m1-pilarnet-smallmask \
  --run.name test \
  --run.wandb-name test-display \
  --run.wandb-project Pretraining-Sonata-PILArNet-M
```

Set `use_wandb=False` to write TensorBoard events under the experiment directory
instead. See {doc}`../hooks/logging`.

## Testing a trained run

```bash
sh scripts/test.sh -c panda/semseg/semseg-pt-v3m2-pilarnet-ft-5cls-enc-upcast-fft \
  -n semseg-pt-v3m2 -w model_best
```

This evaluates the **snapshot** code associated with the experiment. See
{doc}`../evaluation/index`.
